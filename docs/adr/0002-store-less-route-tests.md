# ストアを持たないルートの統合テストは「ナビゲーション配線契約」を守る（ADR 0001 の精緻化）

ストアを所有しないルート（本 PR で扱うメインメニュー `routes/+page.svelte` が該当）の統合テストは、ADR 0001 の「実コンポーネント駆動 + 末端副作用だけスタブ」の原則は引き継ぎつつ、守る契約を **ストア契約からナビゲーション配線契約へ読み替える**。検証するのは ① 必須の `data` prop で描画できる、② 各モードカードが正しいモーダルを開く、③ 各モーダル選択 / ナビリンクが正しい遷移先で `goto` を呼ぶ、の3点。装飾リーフ `KarutaSlideshow`（`requestAnimationFrame` + `Math.random`）は Countdown と同様 `__testmocks__` のスタブで越える。

**理由**: メインメニューは `gameStore` を一切駆動しないため、0001 の「実ストア駆動でストア契約ズレを捕捉」がそのままは適用できない。守るべき退行の本体は「カード→モーダル→遷移先」の配線ズレ（旧テストは特定札練習を `/game?mode=specific` と誤認していた＝まさにこの種の退行）。0001 本文の「#27 も同方針」は、原則（実コンポーネント駆動 + 末端スタブ）を指すのであって実ストア駆動を指すのではない、と射程を補正する。

## Considered Options

- **全ルートを実ストア駆動の型に統一** — 却下。メニューが使わない `gameStore` をテストのためだけにモック/駆動するのは、契約と無関係な手保守を生むだけ。
- **ストアなしルートはビューテストに戻す** — 却下。0001 が廃した「ストア丸ごとモック」への逆戻りで、`data` prop 描画やナビ配線の契約ズレを捕捉できない。

## Consequences

- どの契約を守るかは、ルートが store を駆動するかで決まる。store を所有するルート（`/game` のほか `/settings`＝`settingsStore`、`/statistics`＝`statisticsStore`）は**ストア契約**（ADR 0001 の方針）、所有しないメインメニューは**ナビ配線契約**。`/ranking` 等は対応時に駆動有無を確認して振り分ける。
- メニューのテストは singleton ストアを持たないため、`beforeEach` のストアリセットが不要（`vi.clearAllMocks()` で `goto` スパイを戻すだけ）。`game.spec.ts` の `gameStore.resetSession()` に相当する手当ては要らない。
- `KarutaSlideshow` スタブはこのスペック固有のローカルモック。`gameStore` / IndexedDB / カードデータの旧モックは全廃する（メニュー経路で未使用のため）。
- スコープはナビ配線の退行検知に限定。a11y・レスポンシブ・ローディング/エラーは対象外（到達不能 or 別関心）とし、新規スキップも追跡 Issue も残さない。
